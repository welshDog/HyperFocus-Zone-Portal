# Contributing to HyperFocus Zone Portal

Thanks for being here. This guide keeps contributions clean, safe, and aligned with the ecosystem.

---

## Before you start

1. Read `ECOSYSTEM_HANDOVER.md` — no exceptions
2. Read `PORTAL.md` — understand the repo map
3. Pick **one repo**, **one mission** only
4. Check the repo-level `AGENT-START.md` before changing anything

---

## How to contribute

1. Fork the repo
2. Create a branch — `feat/your-thing` or `fix/your-thing`
3. Make your change
4. Prove it — lint → test → build → THEN commit
5. Open a pull request with a clear title and description
6. Reference any related issue if one exists

---

## Rules

- Never commit `.env` files or secrets
- Never work across multiple repos in one PR
- Keep docs updated after changes
- Respect the sacred rules in `ECOSYSTEM_HANDOVER.md`
- No walls of text in commit messages — short and clear wins

---

## Neurodivergent-first culture

This project is built by and for neurodivergent people.

- Ask questions freely — no question is too basic
- Take breaks, chunk the work, move at your pace
- We celebrate progress, not perfection
